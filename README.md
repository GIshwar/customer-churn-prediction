# \# Customer Churn Prediction

# 

# \## Problem Statement

# 

# Subscription-based telecom businesses lose revenue when customers churn (cancel service). Retention campaigns are expensive, so it's more cost-effective to proactively identify customers likely to churn and intervene early, rather than treating all customers equally.

# 

# This project builds an end-to-end machine learning system to predict customer churn using account, demographic, and service-usage data, with a focus on:

# 

# \- Identifying the strongest drivers of churn (explainability)

# \- Optimizing for business cost, not just accuracy (false negatives — missed churners — are more costly than false positives — unnecessary retention offers)

# \- Deploying the model as a usable API, not just a notebook

# 

# \## Dataset

# 

# Telco Customer Churn dataset (Kaggle, \~7,000 customers), containing demographics, account information, subscribed services, and churn label.

# 

# \## Success Metric

# 

# Primary: Recall on the churn class (catching as many true churners as possible)

# 

# Secondary: Precision, F1, ROC-AUC (to avoid over-flagging too many false positives)
